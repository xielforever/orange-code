# Orange Code 0.1.0 发布说明 (草案)

## 概述

Orange Code `0.1.0` 是当前 Rust 实现的首个公开发布准备里程碑。Orange Code 受 Claude Code 启发，并作为一个**纯净环境（clean-room）**的 Rust 实现进行构建；它并非直接的移植或复制。本次发布的核心在于提供一个可用的本地 CLI 体验：包括交互式会话、非交互式提示（prompts）、工作区工具、配置加载、会话管理、插件机制以及本地代理/技能发现。

## 亮点

- Orange Code 初始公开 `0.1.0` 候选版本
- 以安全的 Rust 实现作为当前主要的产品形态
- `claw` (或可别名为 `orange`) CLI，用于交互式和单次编码代理工作流
- 内置感知工作区的工具，支持：终端 shell、文件操作、搜索、网页抓取/搜索、待办事项（todo）跟踪以及笔记本更新
- 提供用于状态查看、上下文压缩、配置检查、会话管理、差异（diff）/导出和版本信息的斜杠命令（Slash-command）
- 本地插件、代理（agent）和技能的发现/管理界面
- OAuth 登录/注销机制以及模型/提供商选择功能

## 安装与运行

此版本目前主要面向从源码构建：

```bash
cargo install --path crates/claw-cli --locked
# 或者
cargo build --release -p claw-cli
```

运行:

```bash
claw
claw prompt "总结这个存储库"
```

## 已知限制

- 目前仅支持源码构建分发；尚未发布打包好的构建产物
- CI 目前覆盖了 Ubuntu 和 macOS 的发布构建、检查和测试
- 尚未确认对 Windows 的发布就绪状态
- 部分集成测试是可选的（opt-in），因为它们需要真实的提供商凭证和网络访问权限
- 在 `0.x` 发布系列中，公共接口可能会继续演进

## 推荐的发布定位

将 `0.1.0` 定位为 Orange Code 当前 Rust 实现的首个公开发布版本，主要面向那些习惯从源码构建的早期采用者。其功能边界已足够满足实际使用需求，而打包和发布自动化则可以在后续版本中继续完善。

## 本草案的验证依据

- 从 `Cargo.toml` 验证了工作区版本
- 从 `cargo metadata` 验证了 CLI 二进制/包的路径
- 从 `cargo run --quiet --bin claw -- --help` 验证了 CLI 命令表面
- 从 `.github/workflows/ci.yml` 验证了 CI 覆盖率
